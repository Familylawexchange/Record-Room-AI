Deployment modes

Local/private admin mode

Run node server.js on your computer. Originals are stored under ./record-room-data/uploads, extracted text under ./record-room-data/extracted-text, and records in ./record-room-data/database.sqlite.

Document saved to your local Record Room database.

Public website mode

Deploy the same app to a Node host such as Render or Railway. Public submissions stay pending and private until the protected admin dashboard approves publication.

Your submission has been received for review. Submission does not guarantee publication.

Private admin/desktop intake

Upload documents from this computer

Use this local/private workflow for your own document intake. Uploads are private by default and saved to the server-side Record Room database.

Accepted: PDF, DOCX, DOC, TXT, JPG, JPEG, PNG. Maximum size: 25 MB per file. Malware scanning is a placeholder until a scanner is connected.

Public submission portal

Submit a document for review

Public uploads go into a pending review queue. Nothing is published unless an admin approves it and marks it public.

Public search safety

Search approved public records only

Public results only include documents that are both approved and marked public. Pending, rejected, private, admin-only, and redaction-needed documents are excluded.

Admin-only dashboard

Review queue and records

Admin API protection uses an admin token in production. Local development opens in the browser without custom headers; the default local token is printed in the server console and can also be supplied with /admin?token=local-dev-admin.

Local-dev login: use local-dev-admin unless you set RECORD_ROOM_ADMIN_TOKEN. The frontend sends this token automatically for protected admin API calls.
